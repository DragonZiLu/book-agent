# cangjie-skill vs book-agent 系统性对比

## 一、核心定位

| 维度 | cangjie-skill | book-agent |
|------|:---:|:---:|
| 目标 | 把**任意书**蒸馏成一组可独立调用的 Agent Skills | 把**投资书**提炼成一个投资框架 + 直接分析股票 |
| 适用领域 | 通用（已覆盖投资/说服/中医/战略） | 专精（只有投资） |
| 使用者 | AI Agent（被动调用） | 人 + AI Agent（主动分析） |
| 输出形态 | 20+ 个独立 `SKILL.md` + `INDEX.md` | 1 个 `SKILL.md` + 4 个维度文件 + `rules.json` |

> **区别本质**：cangjie 把书拆成**零件箱**（每个 skill 独立可调用），book-agent 把书炼成**一台机器**（框架整体运行）。

---

## 二、方法论对比

| 层级 | cangjie-skill (RIA-TV++) | book-agent |
|------|--------------------------|------------|
| **理解阶段** | 阶段0：Adler 四步分析（结构→解释→批判→应用） | Step1：流派识别（value/growth/quant/macro） |
| **提取策略** | 阶段1：5 个 sub-agent **并行**提取（框架/原则/案例/反例/术语） | Step2：单一 executor **串行逐章**提取 |
| **验证机制** | 阶段1.5：三重验证（跨域佐证+预测力+独特性），通过率 25-50% | Step3：三层评估（L1信息密度+L2真召回+L3闭卷测试） |
| **结构化** | 阶段2：RIA++ 六维构造（引用+释义+案例+触发+步骤+边界） | rules.json 数值规则 + 四维度 Markdown |
| **链接** | 阶段3：Zettelkasten 依赖图 + INDEX.md | ❌ 无（单一框架，不拆分） |
| **测试** | 阶段4：压力测试 + 诱饵题 | L3 ScenarioQA（出题→答题→判分） |
| **反馈闭环** | ❌ 回炉重做是手动触发 | ✅ 自动携带 weak chapter 反馈重试（最多2轮） |

## 三、质量体系深度对比

### cangjie-skill 更强的地方

| 能力 | cangjie | book-agent |
|------|:---:|:---:|
| 预测力验证（V2：能否回答书里没明说的问题） | ✅ | ❌ |
| 独特性验证（V3：排除常识，只保留真正独特的方法论） | ✅ | ❌ |
| 淘汰审计（rejected/ 保留被筛掉的内容，全流程可追溯） | ✅ | ❌ |
| 边界盲点标注（B 段：明确何时不适用） | ✅ | ❌ |
| 诱饵测试（不应触发的场景，防止误调用） | ✅ | ❌ |
| 触发条件明确定义（description 含 trigger，Agent 可自动路由） | ✅ | ❌ |
| 多语言支持（README 中/英/日三语） | ✅ | ❌ |
| 生态咬合（nuwa 蒸馏人 → cangjie 蒸馏书 → darwin 进化） | ✅ | ❌ |

### book-agent 更强的地方

| 能力 | cangjie | book-agent |
|------|:---:|:---:|
| 自动反馈闭环重试（不通过自动回炉） | ❌ | ✅ |
| 量化规则提取（rules.json 可直接代码执行，零幻觉） | ❌ | ✅ |
| 信息密度 L1（纯规则质量检查，零 LLM 成本） | ❌ | ✅ |
| 四态判分（correct / wrong / not_covered / hallucinated） | ❌ | ✅ |
| 置信区间（Wilson CI，处理小样本不确定性） | ❌ | ✅ |
| 闭卷信息瓶颈测试（答题只靠 Skill，不偷看原文） | ❌ | ✅ |
| 成本追踪 + 硬熔断（$5 上限自动停止） | ❌ | ✅ |
| 直接分析标的（advise 子命令，输入指标输出报告） | ❌ | ✅ |
| 进度条 + 并行评估 + 耗时统计 | ❌ | ✅ |

## 五、可互相借鉴的点

| book-agent → 借鉴 cangjie | cangjie → 借鉴 book-agent |
|---------------------------|---------------------------|
| **并行提取**：当前逐章串行提取，可改为多 agent 并行 | **自动反馈重试**：未通过测试自动回炉，不依赖人工判断 |
| **三重验证**：L2 只有召回率，没有预测力和独特性验证 | **闭卷信息瓶颈测试**：测试是否只需 skill 就能答对问题 |
| **边界盲点标注**：当前 Skill 没有"何时不适用"段落 | **成本追踪+熔断**：多 agent 架构更需要成本控制 |
| **触发条件**：Skill 没有 trigger，无法被 Agent 自动路由 | **量化规则驱动**：投资领域天然适合数值阈值提取 |
| **Zettelkasten 链接**：multi-book 场景需要 skill 间关系图 | **四态判分**：区分"答错"和"没覆盖"和"幻觉" |
| **淘汰审计**：保留被筛掉的内容及淘汰原因，可追溯 | **信息密度 L1**：零 LLM 成本的质量检查层 |
| **独立 skill 拆分**：当前单一框架，可拆为独立可调用的模块 | **置信区间**：小样本下的统计严谨性 |

## 七、总结

**cangjie-skill** 是"把书拆成技能零件，让 Agent 随时调用"——通用、严谨、可组合，适合构建 AI 知识库。

**book-agent** 是"把投资书炼成一台分析机器，直接给你买卖建议"——专精、自动化、端到端，适合投资人日常使用。

**两者不是竞争关系，而是上下游关系**：
- cangjie 的输出（独立 SKILL.md）可以作为 book-agent 的输入
- book-agent 的评估器（L1/L2/L3）可以反哺 cangjie 的质量验证
- 最优方案：用 cangjie 的 RIA++ 方法论 + book-agent 的工程自动化能力
